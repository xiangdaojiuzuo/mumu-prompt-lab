# 盤中判斷助手（Android 原型 v0.6｜元大 A71 當沖校正版）

這是一個僅供個人實機測試的 Android 原型：使用 Android 系統的螢幕擷取授權讀取目前可見畫面，以裝置端中文 OCR 擷取股票代號、成交價、開高低與漲跌幅，再用可拖曳的側邊浮窗顯示盤中判斷。

## 目前包含

- 不儲存截圖的持續螢幕擷取
- 約每 0.9 秒執行一次裝置端 OCR
- 側邊浮窗與拖曳、關閉功能
- 股票代號／成交價／開高低／漲跌幅解析
- 依 Samsung A71 的 1080×2400 元大版面，固定辨識上方代號與現價
- 自動分辨分時五檔、逐筆成交、1分K、5分K與下單頁
- 解析均價5／10／20、MACD、K／D，並進行 1分K＋5分K 雙週期確認
- 解析分時頁的一檔買價與一檔賣價
- 蒐集多筆價格後才進行動能判斷
- 可選填持股代號與成本；只對相同代號提供停損、續抱與移動停利提醒
- OCR 資訊不足時顯示「資訊不足」，不產生強行買賣訊號
- 單次 OCR 漏讀時沿用最近 8 秒內的有效資料，不再讓浮窗反覆閃爍資訊不足
- 顯示 0～100 當沖分數與實際讀取欄位，方便辨認 OCR 是否正常
- 納入開盤價、日內位置、短線動能、均線、KD、MACD與五檔委買賣總量
- 移除浮窗的安全截圖旗標，助手運作中仍可使用手機截圖
- OCR 自動排除浮窗覆蓋範圍，避免把自己的判斷文字讀成行情
- 使用固定測試簽章；安裝 v0.4 後，後續版本可直接覆蓋更新
- 記憶同一股票最近 10 分鐘的資訊頁、日K、1分K與5分K資料
- 四個頁面收集完成後，顯示「站上開始偏多」與「跌破開始偏空」的動態價格界線
- 增加日K頁籤與下單頁專用辨識範圍
- 浮窗改為精簡模式：只顯示能否進場、偏多／偏空價格線及一句主要依據
- 不使用 AccessibilityService，不會讀取密碼、不會點擊下單按鈕

## Android Studio 編譯

1. 使用 Android Studio 開啟本資料夾。
2. 等待 Gradle 同步並安裝 Android SDK 35。
3. 以 USB 連接測試手機，或建立 Android 8.0 以上模擬器。
4. 按 Run 安裝 `app`。

## GitHub 自動編譯

專案內含 `.github/workflows/build-apk.yml`。推送到 GitHub 的獨立編譯分支後會自動使用 Java 17、Gradle 8.9 與 GitHub 的 Android SDK 編譯除錯版 APK，成品名稱為 `yuanta-screen-assistant-v0.6-debug`，保留 14 天。也可以在 Actions 頁面手動執行 `Build Android APK`。

第一次啟動需允許：

1. 通知（顯示擷取正在運作）
2. 顯示在其他應用程式上層
3. 系統螢幕分享／擷取

啟動後切換到券商個股報價或 K 線頁面。畫面不會保存到相簿；停止服務或按浮窗的 × 後，擷取會結束。

## 校正版判斷規則

- 至少取得 8 筆同一股票的有效價格後才判斷。
- 價格在開盤價上、擷取區間動能大於 0.20%，且位於當日區間的 55%～92% 時，顯示「偏多進場候選」。
- 價格在開盤價下、擷取區間動能小於 -0.20% 時，顯示「偏空／避免進場」。
- 接近當日最高價但動能不足時，顯示「避免追價」。
- 若輸入成本，虧損達 1.5% 顯示停損警示；獲利達 2% 且短線轉弱時顯示移動停利／減碼。
- 在 K 線頁需依序查看 1分K與5分K：兩者價格、短均線與 KD 同步偏多才顯示進場候選；方向相反時顯示觀望。
- 依序查看資訊頁、日K、1分K與5分K後，以1分／5分當根高低、當日開盤及日K均5計算偏多與偏空觸發價；四頁資料需為同一股票且在10分鐘內。
- 同一股票相鄰兩次辨識價格若跳動超過 15%，或現價與當日高低明顯矛盾，本次 OCR 資料直接捨棄。

這些門檻是為了測試資料流程而設定，不是正式交易策略，也未納入手續費、交易稅、逐筆成交、委買賣隊列或大盤同步。

## 已知限制

- OCR 只能辨識畫面上目前看得見的資料，更新速度不等於券商逐筆行情。
- 已依 2026-07-17 提供的元大深色模式畫面完成第一輪校正；若元大改版、切換淺色模式或調整系統顯示縮放，仍需重新校正。
- K 線畫面若只有圖形而沒有數值，本版不會用像素形狀猜測買賣訊號。
- 螢幕旋轉後應停止再重新啟動擷取。
- 這是原始碼專案，不包含已簽名 APK。
